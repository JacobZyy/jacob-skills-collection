## 1. 概述

Whistle 是基于 Node.js 的 Web 调试代理工具，可拦截、修改 HTTP/HTTPS/WebSocket/TCP 请求和响应。

**核心能力：**
- HTTP/HTTPS/Tunnel/WebSocket 抓包
- 请求/响应内容修改（Header、Body、StatusCode）
- 本地文件替换（Map Local）
- 远程代理转发（Map Remote）
- DNS 劫持
- 限速模拟
- 插件扩展

**访问地址：**
- 桌面客户端：直接启动
- 命令行版本：`http://local.whistlejs.com`
- 默认代理端口：`8899`

**相关仓库：**
- CLI：https://github.com/avwo/whistle
- 客户端：https://github.com/avwo/whistle-client

## 2. 安装与启动

### 安装
```bash
npm i -g whistle
```

### CLI 命令（w2）

| 命令 | 用途 |
|------|------|
| `w2 start` | 后台启动 |
| `w2 start -p 8100` | 指定端口启动 |
| `w2 start --httpsPort 8001` | 启用 HTTPS 代理端口 |
| `w2 start --socksPort 1080` | 启用 SOCKSv5 端口 |
| `w2 start -S storageName` | 使用命名存储目录（多实例） |
| `w2 start -n abc -w 123` | 设置 Web UI 登录 |
| `w2 restart` | 重启 |
| `w2 stop` | 停止 |
| `w2 status` | 查看运行状态 |
| `w2 run` | 前台调试模式 |
| `w2 ca` | 安装根证书 |
| `w2 proxy` | 设置/取消系统代理 |
| `w2 install whistle.xxx` | 安装插件 |
| `w2 uninstall whistle.xxx` | 卸载插件 |
| `w2 add` | 从当前目录 .whistle.js 添加规则（支持 --force） |

### 关键参数

| 参数 | 说明 |
|------|------|
| `-p, --port` | 代理端口（默认 8899） |
| `-P, --uiport` | Web UI 端口（与代理端口分离） |
| `--socksPort` | SOCKSv5 端口 |
| `-S, --storage` | 配置存储目录（多实例必须不同） |
| `-H, --host` | 绑定 IP（默认 INADDR_ANY） |
| `-l, --localUIHost` | Web UI 域名（默认 local.whistlejs.com） |
| `-n, --username` / `-w, --password` | Web UI 用户名密码 |
| `-N, --guestName` / `-W, --guestPassword` | 只读访客账号 |
| `-s, --sockets` | 每个域名最大缓存连接数（默认 256） |
| `-c, --dnsCache` | DNS 缓存时间 ms（默认 60000） |
| `-t, --timeout` | 请求超时（默认 360000ms） |
| `-M, --mode` | 启动模式：pureProxy/debug/multiEnv/capture/disableH2/network/rules/plugins/prod |
| `--no-global-plugins` | 禁用全局插件 |
| `--inspect` | 启用 Node.js 调试器 |

### 多实例运行
```bash
w2 start -p 8010 -S 8010
w2 start -p 8011 -S 8011
```
每个实例需要唯一的端口和存储目录。

### .whistlerc 配置文件
位于 `~/.whistlerc`，支持全局配置和按存储名配置：
```
username=test
password=123
xxx.username=test-storage
*.password=123-default
```

## 3. 规则系统核心

### 3.1 规则语法结构

```
pattern operation [lineProps...] [filters...]
```

| 组件 | 必须 | 说明 |
|------|------|------|
| **pattern** | 是 | URL 匹配表达式，支持域名、路径、通配符、正则 |
| **operation** | 是 | 操作指令，格式 `协议名://操作值` |
| **lineProps** | 否 | 当前行附加配置（仅对当前规则生效） |
| **filters** | 否 | 过滤条件，精确控制规则生效场景 |

### 3.2 规则高级用法

**组合配置** — 单条规则多个操作（顺序执行）：
```
www.example.com/* file:///static-files cache://3600 resCors://*
```

**位置调换** — operation 放在 pattern 前面（多个域名应用相同操作）：
```
proxy://127.0.0.1:8080 www.example.com api.example.com
```
限制：operation 和第一个 pattern 不能同时为 URL/域名格式。

**多行配置** — 使用 `line\`` 换行提高可读性：
```
line`
proxy://127.0.0.1:8080
www.example.com
api.example.com
includeFilter://m:GET
`
```

### 3.3 规则优先级
- 规则从上到下执行
- 后面规则可能覆盖前面规则
- 使用 `lineProps://important` 提升规则优先级

### 3.4 注释
- 使用 `#` 注释行

## 4. 匹配模式（Pattern）

### 4.1 匹配的 URL 类型

| 类型 | 格式 |
|------|------|
| Tunnel 代理 | `tunnel://domain[:port]` |
| WebSocket | `ws[s]://domain[:port]/path?query` |
| HTTP/HTTPS | `http[s]://domain[:port]/path?query` |

### 4.2 域名匹配

格式：`[[schema]://]domain[:port]`

- `//` 前缀表示"使用当前请求的协议"（自动适配 HTTP/HTTPS）
- 支持通配符和 IP 地址

**通配符规则：**

| 通配符 | 等价正则 | 说明 |
|--------|----------|------|
| `*`（域名中） | `/[^/?.]*/` | 单级子域名内零或多个字符 |
| `**`（域名中） | `/[^/?]*/` | 跨级子域名 |
| `*`（端口中） | `/\d*/` | 零或多个数字 |
| `*`（协议中） | `/[a-z]*/` | 零或多个字母 |

**示例：**
```
www.example.com        # 精确域名
*.example.com          # 匹配 www.example.com，不匹配 x.www.example.com
**.example.com         # 匹配 x.y.www.example.com
www.ex*le.com          # 匹配 www.example.com
http*://www.example.com # 同时匹配 http 和 https
www.example.com:8*8   # 匹配端口 88, 8888，不匹配 8080
```

### 4.3 路径匹配

```
www.example.com/path         # 精确路径前缀匹配
www.example.com/path?name=   # 精确路径 + 查询参数（name= 必须存在）
```

路径匹配自动包含子路径：匹配 `/path` 包含 `/path/`、`/path/sub`、`/path/file.html`，但不包含 `/path-other`。

### 4.4 路径通配符（显式 `^` 前缀）

`*` 是合法 URL 字符，作为通配符需加 `^` 前缀：
```
^https://**.example.com/data/*/result?q=*23
```

| 通配符 | 等价正则 | 范围 |
|--------|----------|------|
| `*`（路径中） | `/[^?/]*/` | 单路径段 |
| `**`（路径中） | `/[^?]*/` | 多级路径 |
| `***`（路径中） | `/.*/` | 任意字符 |
| `*`（查询参数中） | `/[^&]*/` | 单个参数值 |
| `**`（查询参数中） | `/.*/` | 跨参数值 |

### 4.5 正则匹配

```
/pattern/[flags]
```
- flags：`i`（忽略大小写）、`u`（Unicode）
- 示例：`/\.test\./`、`/key=value/i`、`/\/api\/v1\/data/i`

### 4.6 子匹配捕获（$0-$9）

```
^http://*.example.com/v0/users/** file:///User/xxx/$1/$2
/regexp\/(user|admin)\/(\d+)/ reqHeaders://X-Type=$1&X-ID=$2
```
- `$0`：完整匹配
- `$1-$9`：捕获组内容

### 4.7 自动路径追加

域名和路径匹配会自动追加原始子路径：
```
https://*.example.com/path/to https://www.test.com/test
# 请求 https://abc.example.com/path/to/x/y → https://www.test.com/test/x/y
```

## 5. 过滤器（Filters）

```
pattern operation includeFilter://pattern1 excludeFilter://patternN ...
# 多过滤器间为 OR 关系
```

| 过滤器 | 匹配对象 | 示例 |
|--------|----------|------|
| `b:pattern` | 请求体 | `includeFilter://b:keyword` / `excludeFilter://b:/regexp/i` |
| `m:pattern` | HTTP 方法 | `includeFilter://m:GET` / `excludeFilter://m:POST` |
| `i:pattern` | 客户端或服务端 IP | `includeFilter://i:192.168` |
| `clientIp:pattern` | 客户端 IP | `excludeFilter://clientIp:10.0.0.1` |
| `serverIp:pattern` | 服务端 IP | `includeFilter://serverIp:/^10\./` |
| `s:pattern` | 响应状态码 | `includeFilter://s:/^20/` / `excludeFilter://s:500` |
| `reqH.header-key:pattern` | 请求头 | `includeFilter://reqH.content-type:json` |
| `resH.header-key:pattern` | 响应头 | `excludeFilter://resH.x-proxy:test` |
| `chance:probability` | 概率匹配 | `includeFilter://chance:0.5` |
| 其他 `xxxxxx` | 请求 URL | `includeFilter://*/cgi-*` / `excludeFilter://www.test.com` |

值可以是关键字（子串匹配）或正则 `/regexp/[i]`。

## 6. 操作指令（Operation）通用规范

### 6.1 值的指定方式

1. **内联值**（括号包裹）：`reqHeaders://x-proxy=Whistle`
2. **内嵌值**（代码块引用）：
   ```
   ``` key.json
   content
   ```
   pattern protocol://{key.json}
   ```
3. **Values 模块引用**：`pattern file://{result.json}`（引用 Values 面板中的内容）
4. **本地文件路径**：`pattern reqHeaders:///User/xxx/filepath`
5. **远程 URL**：`pattern resHeaders://https://example.com/config.json`
6. **临时文件**：按住 Ctrl/Cmd 点击 `protocol://temp.json` 编辑
7. **括号字面值**：`reqHeaders://(/User/xxx/yyy.txt)` 把路径当字符串使用

### 6.2 模板字符串

支持 ES6 模板字符串 `` `...${variable}...` ``：

| 变量 | 说明 |
|------|------|
| `${now}` | Date.now() |
| `${random}` | Math.random() |
| `${randomUUID}` | crypto.randomUUID() |
| `${randomInt(n)}` / `${randomInt(n1-n2)}` | 随机整数 |
| `${reqId}` | 每个请求的 Whistle ID |
| `${url}` | 请求完整 URL |
| `${url.protocol}` / `${url.hostname}` / `${url.host}` / `${url.port}` / `${url.path}` / `${url.pathname}` / `${url.search}` | 解析后的 URL 各部分 |
| `${method}` | HTTP 方法 |
| `${query.xxx}` | 查询参数值 |
| `${reqHeaders.xxx}` / `${resHeaders.xxx}` | 请求/响应头 |
| `${reqCookies.xxx}` / `${resCookies.xxx}` | 请求/响应 Cookie |
| `${statusCode}` | 响应状态码 |
| `${clientIp}` / `${clientPort}` | 客户端 IP/端口 |
| `${serverIp}` / `${serverPort}` | 服务端 IP/端口 |
| `${host}` | Whistle 监听 IP |
| `${port}` | Whistle 监听端口 |
| `${version}` | Whistle 版本号 |
| `${env.xxx}` | process.env.xxx |

### 6.3 数据对象格式

**JSON 格式：** `{"key1": value1, "key2": value2}`

**行格式：**
```
key1: value1
key2:value2
a.b.c: 123
```
等价于 `{"key1":"value1","key2":"value2","a":{"b":{"c":123}}}`

**内联/查询格式：** `key1=value1&key2=value2`

## 7. 操作指令速查表

### 7.1 Map Local（本地文件替换）

| 协议 | 说明 |
|------|------|
| `file://` | 返回本地文件，不请求服务器 |
| `xfile://` | 返回本地文件，文件不存在则请求服务器 |
| `tpl://` | 用本地文件作为模板（支持模板字符串） |
| `xtpl://` | tpl + 文件不存在时请求服务器 |
| `rawfile://` | 返回本地文件原始内容（不自动设置 Content-Type） |
| `xrawfile://` | rawfile + 文件不存在时请求服务器 |

**file 示例：**
```
www.example.com/static file:///Users/xxx/project
# 请求 /static/js/app.js → /Users/xxx/project/static/js/app.js

# 禁用自动路径追加（尖括号）：
www.example.com file://</Users/xxx/index.html>

# 多目录回退搜索（| 分隔）：
www.example.com/static file:///path/to/proj1|/path/to/proj2

# 动态路径（捕获组）：
^www.example.com/user/*/profile file:///mock/profiles/user-$1.json

# 内联 JSON 响应：
www.example.com/api file://({"status":"ok"})
```

### 7.2 Map Remote（远程代理）

| 协议 | 说明 |
|------|------|
| 直接写协议+域名 | 转发到指定 URL |
| `https://` / `http://` | 指定协议转发 |
| `wss://` / `ws://` | WebSocket 转发 |
| `tunnel://` | TCP 隧道转发 |

**URL 转发示例：**
```
www.example.com/path/to www.test.com/test
# 请求 /path/to/x/y → www.test.com/test/x/y
```

### 7.3 DNS 劫持

| 协议 | 说明 |
|------|------|
| `host://` | 修改 DNS 解析（指向 IP 或域名） |
| `xhost://` | host + 解析失败则正常请求 |
| `proxy://` / `http-proxy://` | HTTP 代理转发 |
| `xproxy://` / `xhttp-proxy://` | proxy + 代理失败则正常请求 |
| `https-proxy://` / `xhttps-proxy://` | HTTPS 代理 |
| `socks://` / `xsocks://` | SOCKS 代理 |
| `pac://` | PAC 脚本代理 |

**host 示例：**
```
www.example.com/test1 127.0.0.1           # 保留原端口
www.example.com/test2 host://127.0.0.1:5173
www.example.com/test3 host://www.test.com  # CNAME 效果
```

**proxy 示例：**
```
www.example.com proxy://127.0.0.1:8080
www.example.com proxy://127.0.0.1:8080?host=1.1.1.1:8080  # 上游代理指定 IP
```

**host 与 proxy 优先级：**
- 默认：host 生效，proxy 被忽略
- `enable://proxyFirst` 或 `lineProps://proxyFirst`：只有 proxy 生效
- `enable://proxyHost` 或 `lineProps://proxyHost`：两者同时生效

### 7.4 请求改写（Rewrite Request）

| 操作 | 说明 |
|------|------|
| `method://` | 修改 HTTP 方法 |
| `urlParams://` | 添加/修改查询参数 |
| `delete://urlParams.xxx` | 删除查询参数 |
| `pathReplace://` | 路径替换（支持正则） |
| `sniCallback://` | 动态 TLS 证书选择 |
| `tlsOptions://` | TLS 选项配置 |
| `reqHeaders://` | 修改请求头 |
| `delete://reqHeaders.xxx` | 删除请求头 |
| `forwardedFor://` | 修改 X-Forwarded-For |
| `ua://` | 修改 User-Agent |
| `auth://` | 添加 Authorization 头 |
| `cache://` | 设置缓存时间 |
| `referer://` | 修改 Referer |
| `reqType://` | 修改 Content-Type |
| `reqCharset://` | 修改请求编码 |
| `reqCookies://` | 修改请求 Cookie |
| `reqCors://` | 添加 CORS 头 |
| `reqBody://` | 替换整个请求体 |
| `reqMerge://` | 合并 JSON 到请求体 |
| `reqPrepend://` | 在请求体前插入内容 |
| `reqAppend://` | 在请求体后追加内容 |
| `reqReplace://` | 搜索替换请求体内容 |
| `reqWrite://` | 写入原始请求数据 |
| `reqWriteRaw://` | 写入原始请求数据（不做编码转换） |
| `reqRules://` | 对匹配请求应用子规则 |
| `reqScript://` | 用脚本修改请求 |

**常用示例：**
```
# 修改方法
www.example.com/path method://post

# 添加请求头
www.example.com reqHeaders://{"X-Custom-Header":"test"}
www.example.com reqHeaders://x-proxy=Whistle

# 修改 User-Agent
www.example.com ua://Mozilla/5.0...

# 替换请求体
www.example.com/api reqBody://({"newData":"value"}) method://post

# 合并参数
www.example.com/api urlParams://({"cmdname":"Test"}) delete://urlParams.oldParam
```

### 7.5 响应改写（Rewrite Response）

| 操作 | 说明 |
|------|------|
| `statusCode://` | 直接返回指定状态码（不请求服务器） |
| `replaceStatus://` | 替换服务器返回的状态码 |
| `redirect://` | 302 重定向 |
| `locationHref://` | 修改 Location 响应头 |
| `resHeaders://` | 修改响应头 |
| `delete://resHeaders.xxx` | 删除响应头 |
| `responseFor://` | 按条件返回不同响应 |
| `resType://` | 修改响应 Content-Type |
| `resCharset://` | 修改响应编码 |
| `resCookies://` | 修改响应 Cookie |
| `attachment://` | 添加 Content-Disposition 下载头 |
| `resCors://` | 添加 CORS 响应头 |
| `resBody://` | 替换整个响应体（请求仍发到服务器） |
| `resMerge://` | 合并 JSON 到响应体 |
| `resPrepend://` | 在响应体前插入内容 |
| `resAppend://` | 在响应体后追加内容 |
| `resReplace://` | 搜索替换响应体内容 |
| `htmlPrepend://` / `htmlBody://` / `htmlAppend://` | HTML 插入 |
| `cssPrepend://` / `cssBody://` / `cssAppend://` | CSS 插入 |
| `jsPrepend://` / `jsBody://` / `jsAppend://` | JavaScript 插入 |
| `trailers://` | HTTP Trailers |
| `resWrite://` / `resWriteRaw://` | 写入原始响应数据 |
| `resRules://` / `resScript://` | 响应子规则/脚本 |
| `frameScript://` | WebSocket 帧脚本 |

**常用示例：**
```
# 修改状态码
www.example.com statusCode://500
www.example.com replaceStatus://200

# 添加响应头
www.example.com resHeaders://{"X-Custom":"value"}
www.example.com resHeaders://x-proxy=Whistle
www.example.com resCors://*

# 重定向
www.example.com/old redirect://https://www.example.com/new

# 替换响应体
www.example.com/api resBody://({"status":"modified"})

# HTML/CSS/JS 注入
www.example.com htmlBody://<script>console.log('injected')</script>
www.example.com jsAppend://console.log('end')
www.example.com cssBody://body { background: red; }

# 状态码过滤
www.example.com resBody://({"error":"ServiceUnavailable"}) includeFilter://s:503
```

### 7.6 通用规则

| 操作 | 说明 |
|------|------|
| `pipe://` | 数据流管道处理 |
| `delete://` | 删除特定内容 |
| `headerReplace://` | 灵活的请求/响应头替换 |

### 7.7 限速

| 操作 | 说明 |
|------|------|
| `reqDelay://` | 请求延迟（ms） |
| `resDelay://` | 响应延迟（ms） |
| `reqSpeed://` | 请求速度限制（KB/s） |
| `resSpeed://` | 响应速度限制（KB/s） |

### 7.8 调试工具

| 操作 | 说明 |
|------|------|
| `weinre://` | 远程 DOM 调试 |
| `log://` | 远程日志捕获 |

### 7.9 设置

| 操作 | 说明 |
|------|------|
| `style://` | 样式注入 |
| `enable://` | 启用特定功能（如 capture, proxyHost, proxyFirst） |
| `disable://` | 禁用特定功能（如 capture, h2, auto2http） |
| `lineProps://` | 设置当前行属性（如 important, safeHtml, proxyHost, weakRule） |
| `excludeFilter://` / `includeFilter://` | 请求过滤 |
| `ignore://` / `skip://` | 忽略/跳过匹配请求 |

### 7.10 file vs resBody 区别

| 方面 | file | resBody |
|------|------|---------|
| 是否请求服务器 | 否 | 是（先请求，再替换） |
| 响应 | 返回本地内容，200 | 替换服务器响应体 |

## 8. HTTPS 配置

### 安装根证书
1. 点击 Whistle 界面顶部 **HTTPS** 按钮
2. 用手机扫描 QR 码下载（或 PC 端直接在浏览器中打开 `http://local.whistlejs.com`）

**直接下载地址：** `http://[PC-IP]:[端口]/cgi-bin/rootca`

**iOS：**
- 设置 → 通用 → VPN 与设备管理 → 安装
- 设置 → 通用 → 关于本机 → 证书信任设置 → **启用完全信任**

**Android：**
- 设置 → 安全 → 加密与凭据 → 安装证书 → CA 证书
- Android 12+：更多安全设置
- 华为 EMUI：先关闭"纯净模式"

### 移动端抓包
1. 安装根证书（必须）
2. 配置 Wi-Fi 代理为手动 → 输入服务器 IP 和 Whistle 端口（默认 8899）

**成功标志：**
- 手机能正常访问网页
- Whistle 能捕获 HTTPS 请求
- 无安全警告

## 9. 插件系统

### 9.1 插件开发

插件是 npm 包，前缀 `whistle.`。使用 `lack` 脚手架：

```bash
npm i -g lack   # >= 1.4.0
lack init <type>
lack watch      # 开发模式自动挂载
```

**插件文件类型：**

| 文件 | 用途 | 作用阶段 |
|------|------|----------|
| `rules.txt` | 全局静态规则 | 插件安装时加载 |
| `_rules.txt` | 协议作用域请求规则 | 匹配插件协议时 |
| `resRules.txt` | 协议作用域响应规则 | 响应阶段 |
| `src/auth.ts` | 请求认证 | 请求认证 |
| `src/sniCallback.ts` | 动态 TLS 证书 | TLS 握手 |
| `src/rulesServer.ts` | 动态规则生成 | 请求阶段 |
| `src/tunnelRulesServer.ts` | Tunnel 规则生成 | Tunnel 阶段 |
| `src/resRulesServer.ts` | 动态响应规则 | 响应阶段 |
| `src/statsServer.ts` | 只读请求监控 | 请求阶段 |
| `src/resStatsServer.ts` | 只读响应监控 | 响应阶段 |
| `src/server.ts` | 完整代理处理器 | 全流程 |
| `src/pipe.ts` | 数据管道转换 | 数据流 |

**auth 接口：**
```ts
export default async (req: Whistle.PluginAuthRequest, options: Whistle.PluginOptions): Promise<boolean>
```
返回 `true` 放行，`false` 拒绝（403）。可用 `req.setHtml()`、`req.setLogin()`、`req.setRedirect()`、`req.setHeader()`。

**sniCallback 接口：**
```ts
export default async (req: Whistle.PluginSNIRequest, options: Whistle.PluginOptions): Promise<boolean | { key: string; cert: string }>
```
返回 `true` 用内置证书，`false` 不解密，`{ key, cert }` 用自定义证书。

**插件 UI：** 支持 Tab 模式、Modal 模式、新标签模式。配置在 `package.json` 的 `whistleConfig` 中。

### 9.2 whistleConfig 配置项

| 配置 | 说明 |
|------|------|
| `hideLongProtocol` | 隐藏 `whistle.plugin://` |
| `hideShortProtocol` | 隐藏 `plugin://` |
| `priority` | 排序优先级（数字越小优先级越高） |
| `favicon` | 插件图标 |
| `noOption` | 无管理 UI 时灰化按钮 |
| `enableAuthUI` | 对插件 UI 也启用认证 |
| `pluginHomepage` | 外部主页 URL |
| `openInModal` | 模态框打开配置 |
| `networkColumn` | Network 列表自定义列 |
| `inspectorsTab` / `composerTab` / `toolsTab` | 详情面板自定义标签 |
| `networkMenus` / `rulesMenus` / `valuesMenus` / `pluginsMenus` | 自定义右键菜单 |

## 10. FAQ 常见问题

### 抓包显示 "Tunnel to"
- 纯 TCP 连接（无法解析为 HTTP/HTTPS）
- 未启用 HTTPS：安装根证书并开启 HTTPS
- SSL 解密错误（captureError）
- 请求直接使用 IP 地址（缺少 SNI）：配置 `ip[:port] enable://capture`

### 手机无法抓包
- iOS：确认"完全信任"已启用（设置→通用→关于本机→证书信任设置）
- Android：确认证书安装在正确的 CA 证书位置；检查 SSL Pinning
- 公司应用：Android 需配置 `network_security_config.xml`

### Firefox HTTPS 无法抓包
- 安装证书到 Firefox 或设置 `security.enterprise_roots.enabled` 为 `true`

### mTLS（双向认证）
- 使用 `@clientCert://` 规则配置

### 查看日志
- Network → Tools → Server（运行时错误）
- `~/.WhistleAppData/whistle.log`（崩溃日志）

### 多选规则
- Rules 界面 → Settings → 勾选 "Use multiple rules"

### 过滤轮询请求
- 右键请求 → Settings / Exclude All Matching Hosts（或 URLs）

### 更新版本
```bash
npm i -g whistle && w2 restart
npm i -g whistle --registry=https://registry.npmmirror.com && w2 restart
```

## 11. 调试技巧

1. **逐步验证：** 从简单规则开始，逐步添加复杂条件
2. **日志查看：** Network → Overview 面板查看规则匹配情况
3. **浏览器调试：** 配合浏览器 DevTools 验证实际效果
4. **临时禁用：** 使用 `#` 注释暂时禁用规则
5. **规则未生效时检查：** pattern 是否正确、是否被高优先级规则覆盖、过滤器条件是否满足
